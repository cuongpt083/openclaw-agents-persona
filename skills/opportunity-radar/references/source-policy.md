# Source Policy

Use this source order by default:
1. Paper, technical blog, GitHub, conference talk
2. Analysis from funds, consulting firms, and specialist industry media
3. Industry reports and research houses
4. Official sources from companies, banks, fintechs, and regulators

## Notes

- Start from technical signals when the user wants early opportunities.
- Before elevating a technical signal into a meaningful opportunity, try to confirm it with at least one market-facing or ecosystem source.
- Prefer sources from the last 90 days unless the user explicitly asks for historical context.
- When the topic is geographically sensitive, prioritize sources tied to:
  - China
  - Hong Kong
  - Singapore
  - Thailand
  - Indonesia
  - South Korea

## Signal Tests

A signal is stronger when:
- multiple independent sources point to the same shift,
- there is evidence of actual productization, experimentation, or public rollout,
- the signal maps cleanly to a Vietnamese use case.

A signal is weaker when:
- it appears in only one source,
- it is purely aspirational or marketing-led,
- there is no plausible path to adoption in Vietnam.
